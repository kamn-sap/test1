# テスト項目作成アプリ

1. テスト設計・作成機能（コア機能）
- テストケースの構造化・階層化: テスト項目（大・中・小項目）を論理的・構造的に整理し、ツリー構造やフォルダ分けで管理できる機能。
- 入力項目（必須・任意）の定義: 「テストの目的」「前提条件」「テスト手順」「期待される結果」「入力データ」を明確に記述するテンプレート機能。
- 再利用性とクローニング: 類似したテストケースをコピー、編集、テンプレート化し、再利用する機能。
- 因子・水準を組み合わせたペアワイズ法（オールペア法）のデシジョンテーブル生成
- バージョン管理: テストケースの変更履歴を記録し、必要に応じて以前のバージョンに戻せる機能。 

2. テスト実施・管理機能
- 実施状況の可視化: 「未実施」「成功(Pass)」「失敗(Fail)」「ブロック」など、ステータスを一目で把握できるダッシュボード機能。
- エビデンス（画像・ログ）の添付: スクリーンショットや操作ログを直接テストケースに保存できる機能。
- 担当者アサイン: テスト項目を特定のメンバーに割り当て、進捗を管理する機能。 

3. AI・自動化・効率化機能
- AI生成機能: 要件定義書や画面設計書から自動でテストケースを生成するAI補助機能(githubのcopilot)
- 自動テスト連携: 外部の自動化ツール（Selenium, Appiumなど）と連携し、手動テストの結果と自動化された結果を統合して管理する機能。
- データのパラメータ化: テストデータをCSVなどでインポートし、同じケースを使って複数のデータパターン（境界値など）を効率的にテストできる機能。 

4. 連携・運用機能
- プロジェクト管理ツール連携: redmineで起票したチケットと連携。不具合はGitHubのissuesに起票しテストケースに反映できる機能。
- インポート・エクスポート: ExcelやCSVからテスト項目を一括で読み込み、または書き出せる機能。
レポート機能: テストカバレッジ、不具合の傾向、進捗率を自動生成し、関係者へ共有できる機能。 

5. システム構成

| 役割           | 採用サービス                        | 特徴・役割                                                                 |
|----------------|-------------------------------------|----------------------------------------------------------------------------|
| ホスティング   | Amazon S3 + CloudFront              | 高速なコンテンツ配信とセキュアな静的サイト公開。React を配信。            |
| 認証・認可     | Amazon Cognito                      | ユーザーログイン、チームごとの権限管理を実現。                             |
| API            | AppSync (GraphQL)                   | テスト項目のリアルタイム更新や複雑なリレーション管理に最適。               |
| ロジック実行   | AWS Lambda                          | デシジョンテーブル生成や組み合わせ計算（ペアワイズ法等）を実行。          |
| データベース   | Amazon DynamoDB                     | スケーラブルなNoSQL。テストケースの保存・検索に利用。                     |
| ファイル保存   | Amazon S3                           | テストのエビデンス画像や出力したExcel/CSVファイルの保管。                 |

## 🚀 クイックスタート

ローカル環境でアプリケーションを実行するための手順です。

### 前提条件

- Docker & Docker Compose がインストールされていること
- Node.js 18.x 以上
- npm

### セットアップ手順

#### **ステップ 1: LocalStack を起動**

```bash
cd /workspaces/test1
docker-compose up -d
```

起動状況確認：
```bash
docker-compose logs -f localstack
```

LocalStack が完全に起動するまで 30～60 秒待ちます。以下が表示されたら完了です：
```
Ready to accept connections
```

---

#### **ステップ 2: インフラをデプロイ**

別のターミナルで以下を実行：

```bash
cd /workspaces/test1/infra
npm install
npm run build
npm run deploy:local
```

デプロイ完了時に以下のような出力が表示されます：
```
✓ Test1Stack
Resources:
  ...
GraphQLEndpoint: https://xxxx.appsync-api.us-east-1.amazonaws.com/graphql
```

---

#### **ステップ 3: React アプリを起動**

別のターミナルで以下を実行：

```bash
cd /workspaces/test1/app
npm install
npm start
```

ブラウザが自動で開き、`http://localhost:3000` でアプリケーションが起動します。

---

### ✅ 動作確認

アプリケーションが正常に起動すると、以下の3つのタブが表示されます：

- 📋 **テストケース一覧** - 作成されたテストケースの管理
- ➕ **新規作成** - 新しいテストケースの作成フォーム
- 🔀 **ペアワイズ生成** - オールペア法によるテスト組み合わせ生成

### 各サービスへのアクセス

| サービス | URL | 用途 |
|---------|-----|------|
| React アプリ | http://localhost:3000 | テスト管理 UI |
| DynamoDB Admin | http://localhost:8001 | データベース確認 |
| LocalStack | http://localhost:4566 | AWS サービスエミュレーション |
